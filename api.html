We've kept the domain model for Simba as simple as possible. Everything in the domain is targetted to Users, Roles and Policies.

The User entity is the representation of a principal in Simba.

Set roles = new HashSet () Set sessions = new HashSet ();
userName The unique user name for the user
password The password for the user. Passwords in Simba are salted and hashed before storing them.
name The real surname for the user.
firstName The real first name for the user.
status The current status of the user. The status can be active, inactive or blocked.
inactiveDate The date at which the user has become inactive.
successURL The URL to redirect the user to on successful log on. Simba will only use this URL if the user did not try to access a secured application before bying directed to the login page.
changePasswordOnNextLogon Is the user required to change his password at next login. A user is required to change his password if the password has been used for longer then a certain amount of days, or if an administrator explicitly flags the user as required to change his password.
passwordChangeRequired Is the user ever required to change his password. Certain system accounts can be excluded from the need to change the password.
databaseLoginBlocked Is the user allowed to login using the credentials stored in the Simba database. This can be used if there is another way of authentication in place (eg. Active Directory, ...) and we don't want the user to have multiple credentials
dateOfLastPasswordChange The date when the user last changed his password. This is used to determine when the user should be forced change his password again.
invalidLoginCount The number of successive invalid logins made by the user. This will be used to block out a user, making brute-force attacks impossible.
